Cos'è SmartReporting

SmartReporting è la piattaforma di reportistica automatizzata MSSP di EKO365. Genera report mensili e trimestrali per 9 clienti su 6 linee di servizio, con un control plane centralizzato (supervisor) che orchestra le pipeline e produce un digest mensile aggregato. Tutti gli artefatti finiscono su Wasabi S3; un CLI dedicato (eko365-download) permette il pull selettivo dei report con tre layout di organizzazione.

  • Legge i manifest da s3://eko365-pipelines/_runs/
  • Cross-referenzia con pipeline_registry.py (5 monthly + 1 quarterly attese)
  • Costruisce uno stato aggregato DigestState
  • Renderizza un'email HTML branded EKO365 (Century Gothic, viola/arancio)

CLI eko365-download

Download manifest-aware con filtri (period, pipeline, client, since) e tre layout di organizzazione output. Default folder: ~/EKO365-Reports/.

Recipes

# Archiviazione mensile (default = layout archive)
eko365-download --period 2026-04
# → ~/EKO365-Reports/2026-04/<pipeline>/<client>/<file>

# Condivisione singolo cliente
eko365-download --period 2026-Q1 --client KSGR --layout client --output ~/EKO365-Share/
# → ~/EKO365-Share/KSGR/2026-Q1/<pipeline>/<file>

# Review interno cross-client di una pipeline
eko365-download --period 2026-Q1 --pipeline smartdesk --layout pipeline --output ~/work/
# → ~/work/smartdesk/2026-Q1/<client>/<file>

Layouts

LayoutPath patternUse case

pipeline<pipeline>/<period>/<client>/<file>Review interno di una pipeline

Operatività mensile

Workflow tipico del 5° giorno del mese

  1. Le pipeline schedulate completano i run nei giorni 1-3 (vedi calendario)
  2. 5° del mese 10:00 UTC: cron supervisor → email digest a fabio@eko365.it
  3. Apri l'email, verifica subject (es. 5/5 OK · 0 missing) e badge
  4. Se tutto OK: eko365-download --period 2026-04 per scaricare l'archivio
  5. Per condividere con un cliente: eko365-download --period 2026-04 --client KSGR --layout client --output ~/EKO365-Share/

Cosa fare se vedi MISSING/FAILED

SintomoSignificatoAzione

Trigger manuale di una pipeline

gh workflow run "Infrastructure Pipeline" --repo eko365sb/infra-reports --ref main \
  -f month=2026-04 -f phase=all -f send_email=false

Importante: send_email di default è false (silent on success). Lascia così salvo casi specifici di test.

Lessons learned & pattern

Pattern: workflow integration testing
Verificare sempre che lo script Python modificato sia effettivamente invocato dal workflow di produzione. Modificare main() di un file che il workflow non chiama produce dead code. Bug F4.2 (1 maggio 2026) causato esattamente da questo. Vedi workflow-integration-pattern.md.
Pattern: discover_reports + aws s3 sync
discover_reports(s3_prefix="...") deve usare il parent del s3_destination, non il prefix con period appended. La lib v0.1.4 usa path.relative_to(directory). Bug F4.1 fixato il 30 aprile 2026.
Pattern: gh secret set
Sempre gh secret set NAME --body "$VAL" inline. Mai stdin pipe (corrompe valori).
