Architettura Sorgente

La pipeline sfrutta un approccio email-based unico: Veeam Enterprise Manager (server on-prem EKOBKP08) invia report giornalieri via SMTP alla shared mailbox pipeline-ingest@eko365.it. L'app Entra ID EKO365-Pipeline-MailReader (client credentials, secret valido fino 02/2028) legge le email tramite Graph Mail API con Application Access Policy restrittiva.

Parsing HTML

Il parser gestisce sia formato nativo Veeam (hex color) che forward Outlook (rgb format). Estrae 9 colonne per task: resource_name, status, start/end_time, performance_rate, size, read, transferred, duration. Supporto formato europeo (virgola decimale: "1,1 TB").

Scheduling Completo

0 3 1 * * 1º del mese, 03:00 UTC — M365 Graph full collection (tutti i moduli) M365
0 6 2 * * 2º del mese, 06:00 UTC — SmartBackup AFI fetch + analyze + report; in mesi 03/06/09/12 auto quarter-end: report per-tenant + CTO overview SmartBackup
0 6 3 * * 3º del mese, 06:00 UTC — Desk365 fetch + analyze + report; in mesi 03/06/09/12 auto quarter-end: report per-client PPTX + DOCX + CSV ticket orfani Desk365
0 4 * * 0 Ogni domenica, 04:00 UTC — M365 Graph weekly (Security + Health) M365
0 10 * * * Ogni giorno, 10:00 UTC (12:00 IT) — Veeam fetch + parse. Il 2º del mese auto-escalation a full pipeline (analyze + report); quarter-end detection per report client Veeam
0 14 3 * * 3º del mese, 14:00 UTC — M365 Graph report generation (ops report mensile; in mesi 03/06/09/12 anche report client DOCX + PPTX) M365
Quarter-End Auto-Detection — Tutte e 4 le pipeline rilevano automaticamente i mesi di fine trimestre (03/06/09/12). Nei run trimestrali, oltre ai report operativi interni, vengono generati report client-facing aggiuntivi (PPTX, DOCX per-tenant) e il periodo di analisi viene esteso all'intero trimestre.
